# 抖音B站直播监控节点看板

一个专门为 **抖音 / B站直播监控** 打造的节点化数据看板。

它不是简单的“搜一下直播间列表”，而是把 **关键词监控、节点留档、当前节点推送、跨时间段汇总、黑名单过滤、统一平台配置** 全部揉进一个页面里，直接把“人工盯盘 + 人工统计 + 人工发群”的低效流程砍掉一大截。

> 如果你要的是一个能长期盯直播、按时间节点沉淀数据、还能随时汇总推送到企业微信的工具，这个项目就是干这个的。

---

## 项目亮点

### 1. 节点化监控，不再是老式“几轮几轮”

- 每次抓取结果都按 **节点时间** 保存
- 页面展示的是 **当前节点数据**
- 导出文件里每个节点都有独立 sheet
- 状态、导出、推送、汇总，全部统一使用“节点”口径

### 2. 当前节点自动推送，信息够新、够轻、够直接

- 抓取完成后默认只推送 **当前节点**
- 不再每次都塞整段历史，避免文件臃肿、群里刷屏
- 企业微信适合看“此刻发生了什么”，这个项目就是按这个思路设计的

### 3. 节点区间立即汇总，复盘效率直接拉满

- 支持手动选择 **起始节点** 与 **结束节点**
- 一键执行 **立即汇总并推送**
- 特别适合做：
  - 昨天 17:00 到今天 17:00 的日报
  - 某场活动期间的直播表现汇总
  - 某个监控时段的临时复盘

### 4. 抖音 + B站 统一配置，不再来回切平台

- 平台改成 **统一勾选**
- 关键词、抓取间隔、推送、汇总时间、黑名单，全部只配一次
- 公共操作界面合并，流程更短，误触更少

### 5. 保存后锁定配置，防止误触

- 点 **保存统一配置** 后，页面配置区会自动锁定
- 想改参数时，再点 **重新配置** 解锁
- 对长时间运行的监控场景非常重要，尤其适合多人共用或长期开着页面

### 6. 黑名单过滤，直接跳过不想看的作者

- 支持输入 **作者主页链接** 作为黑名单
- 命中后不再进入监控、当前节点、汇总和导出
- 对“固定不关心作者 / 干扰样本 / 噪音直播间”特别实用

### 7. 抖音 Cookie 支持手动 + 自动获取双模式

- 支持直接粘贴整段 Cookie
- 支持自动打开浏览器扫码登录，再一键刷新获取 Cookie
- 降低 Cookie 获取和维护成本

## 页面现在长什么样

项目当前页面结构是：

- **统一配置区**
  - 选择平台
  - 配置关键词
  - 设置抓取间隔
  - 设置汇总时间
  - 设置汇总时间段
  - 配置企业微信 Webhook
  - 维护黑名单作者
- **节点汇总区**
  - 选择起始节点
  - 选择结束节点
  - 立即汇总并推送
- **抖音 Cookie 区**
  - 手动保存 Cookie
  - 自动获取 Cookie
  - 刷新当前 Cookie 状态
- **平台概览区**
  - 运行状态
  - 节点数
  - 当前房间数
  - 累计不同直播间数
  - 最近推送状态
  - 最近错误
- **当前节点数据区**
  - 分平台展示当前最新节点的数据表格

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

抖音签名相关能力依赖 Node.js 运行环境。

如果本机未安装 Node.js，请先安装并加入 PATH。

### 2. 准备 Cookie

#### 抖音

支持两种方式：

- 直接把整段 Cookie 粘贴到页面里保存
- 使用页面中的“自动获取 Cookie”功能，扫码登录后刷新获取

也可以手动写入：

- `../live/douyin_cookie.txt`
- 或环境变量 `DY_COOKIE`
- 或环境变量 `DOUYIN_COOKIE`

#### B站

- 继续沿用仓库内原有 B站 Cookie 读取逻辑
- 会自动从 `_2_bilibili/live/cookie*.txt` 等位置读取

### 3. 启动服务

```bash
python app.py
```

### 4. 打开页面

浏览器访问：

```text
http://127.0.0.1:5056
```

## 使用方式

### 统一监控

1. 勾选要监控的平台
2. 输入关键词
3. 设置抓取间隔
4. 设置汇总时间与汇总区间
5. 填写企业微信 Webhook
6. 维护黑名单作者主页
7. 点 **保存统一配置**
8. 点 **启动所选平台监控**

### 当前节点推送

- 监控运行后，每个节点抓取完成会自动推送当前节点
- 也可以点击 **立即推送当前节点** 手动推送

### 节点区间汇总

- 在“节点汇总”区域选择起始节点和结束节点
- 点击 **立即汇总并推送**
- 系统会生成该区间内的汇总文件并推送到企业微信

---

## 关于“17:00 到次日 17:00”的配置

如果你要汇总：

- **前一天 17:00 到今天 17:00**

直接这样配置：

- `汇总时间`：`17:00`
- `汇总起始节点时间`：`17:00`
- `汇总结束节点时间`：`17:00`

页面会显示为：

```text
汇总区间 17:00（前一天）~ 17:00（当天）
```

这表示系统会把这 24 小时内的节点纳入自动汇总窗口。

## 注意事项

- 抖音接口更容易受到风控，抓取间隔不要过短，建议 `>= 5 分钟`
- Cookie 失效、验证页、接口静默限制，都会影响抖音结果
- 企业微信 Webhook 地址格式必须正确
- 黑名单使用的是作者主页链接，建议填完整链接，避免误判
- 节点越多、汇总区间越长，导出文件也会越大

这个项目不是一个“看一眼数据”的小脚本。

它更像一个专门服务于 **直播监控、数据节点留档、阶段汇总推送、企业微信播报** 的轻量看板系统。

一句话总结：

> 把直播监控从“人工刷新  + 人工汇总 + 人工发群”，升级成“节点化抓取 + 自动沉淀 + 一键汇总推送”。
